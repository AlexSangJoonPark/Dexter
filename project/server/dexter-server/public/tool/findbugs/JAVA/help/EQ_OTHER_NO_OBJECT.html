Checker Code
  EQ_OTHER_NO_OBJECT
Description
  equals() method defined that doesn't override equals(Object)
Severity
Critical
Detector / Bug Pattern
FindHEmismatch / Eq
More Information

This class defines an equals() method, that doesn't override the normal equals(Object) method defined in the base java.lang.Object class.  Instead, it inherits an equals(Object) method from a superclass. The class should probably define a boolean equals(Object) method.

Bad Code 
public class Vehicle {
    private Sample sample;
    ...
    public boolean equals(Object obj) {
        ... // Checks equality with this Vehicle object with argument object
    }
}

public class Cycle {
    private Sample sample;
    ...
    public boolean equals(Cycle cycle) {
        ... // Checks equality between two cycle object
    }

	// Inconsistent Result : Class Inherits Vehicle.equals(Object obj) from super class
	// And this inherited method will be called if a object of class "Car extends Vehicle" is passed
	// when " new Cycle().equals(new Car());" will be invoked.
}
Good Code 
public class Vehicle {
    private Sample sample;
    ...
    public boolean equals(Object obj) {
        ... // Checks equality with this Vehicle object with argument object
    }
}

public class Cycle {
    private Sample sample;
    ...
    public boolean equals(Cycle cycle) {
        ... // Checks equality between two cycle object
    }
	public boolean equals(Object obj) {
        ... // Checks equality with this Cycle object with argument object
    }
}
