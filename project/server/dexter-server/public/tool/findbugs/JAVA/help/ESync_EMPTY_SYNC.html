More Information

The code contains an empty synchronized block:

synchronized() {}

Empty synchronized blocks are far more subtle and hard to use correctly than most people recognize, and empty synchronized blocks are almost never a better solution than less contrived solutions.

Bad Code 
// The following code attempts to synchronize on an object, but does not execute anything in the synchronized block. 
// This does not actually accomplish anything and may be a sign that a programmer is wrestling with 
// synchronization but has not yet achieved the result they intend.

//However,The reader should not be fooled into thinking that any empty synchronization block can be removed. Please check
http://www.cs.umd.edu/~pugh/java/memoryModel/archive/2183.html

synchronized(this) { }
Good Code 
// The synchronized statement should contain procedures that access or modify data that is exposed to multiple threads
public void setID(int ID){
synchronized(this){
this.ID = ID;
}
}
