Checker Code
  IC_SUPERCLASS_USES_SUBCLASS_DURING_INITIALIZATION
Description
  Superclass uses subclass during initialization
Severity
Critical
Detector / Bug Pattern
FindPuzzlers / IC
More Information

During the initialization of a class, the class makes an active use of a subclass. That subclass will not yet be initialized at the time of this use. For example, in the following code, foo will be null.

Bad Code 
public class CircularClassInitialization {
    static class InnerClassSingleton extends CircularClassInitialization {
        static InnerClassSingleton singleton = new InnerClassSingleton();
    }

    static CircularClassInitialization foo = InnerClassSingleton.singleton; // Incorrect : To use before Initialization and object created
}
Good Code 
public class CircularClassInitialization {
    static class InnerClassSingleton extends CircularClassInitialization {
        static InnerClassSingleton singleton = new InnerClassSingleton(); // Correct : To use once Initialization is done and object created
    }

    static CircularClassInitialization foo = null;

	public void setCircularClassInitialization()
	{
        foo = InnerClassSingleton.singleton; ??
	}
}
