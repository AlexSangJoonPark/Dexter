Checker Code
  CN_IDIOM_NO_SUPER_CALL
Description
  clone method does not call super.clone()
Severity
Critical
Detector / Bug Pattern
CloneIdiom / CN
More Information

This non-final class defines a clone() method that does not call super.clone(). If this class ("A") is extended by a subclass ("B"), and the subclass B calls super.clone(), then it is likely that B's clone() method will return an object of type A, which violates the standard contract for clone().

If all clone() methods call super.clone(), then they are guaranteed to use Object.clone(), which always returns an object of the correct type.

Bad Code 
class House implements Cloneable
{
  private int numberOfRooms;
  private int squareFeet;
   public Object clone()
  {
    return new House(); //Incorrect to not call super.clone() at the first statement;
  }
}
Good Code 
class House implements Cloneable
{
  //As before...
  public Object clone()
  {
    try {
      return super.clone(); // Correct Implementation
    }
    catch (CloneNotSupportedException e) {
      throw new InternalError();
    }
  }
}
