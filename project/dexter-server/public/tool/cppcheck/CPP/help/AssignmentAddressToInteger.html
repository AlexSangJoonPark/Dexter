 //How come you can assign an address to an integer variable like this,the complier will not give an error.

Good Code 
/*The number 0x28ff1c is just the hexadecimal (base-16) representation of the decimal (base-10) number 2686748.
As cout defaults to printing decimal values for integers, that is probably the number you got printed.
The case with char b = 0x28ff1c is slightly different, because 
char is not large enough to hold that value. The practical result is that it gets truncated to 0x1c. 
cout treats char specially, because it is normally used to hold textual data, so cout prints the character 
that has the code 0x1c, which is some kind of control character. You could try it with 0x41 for example
(which represents 'A' in ASCII and UTF-8).
*/
